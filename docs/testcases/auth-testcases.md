## POST /register

### TC-REG-001 — Успешная регистрация с валидными данными

**Тип:** Позитивный
**Предусловие:** Сервер запущен. Пользователь testuser не существует в БД.

**Шаги:**
1. POST на /api/auth/register
2. Заголовок: Content-Type: application/json
3. Body: { "name": "testuser", "password": "StrongPass123" }
4. Нажать Send

**Ожидаемый результат:**
- Статус: 201 Created
- Body содержит id пользователя
- Поле password отсутствует в ответе

**Фактический результат:**
- Статус: 201 Created
- Body: { "message": "User created successfully" } — id не возвращается

**Статус: FAIL** — см. BUG-003

## POST /login

### TC-LOG-001 — Успешный логин с валидными данными

**Тип:** Позитивный
**Предусловие:** Пользователь testuser зарегистрирован.

**Шаги:**
1. POST на /api/auth/login
2. Body: { "name": "testuser", "password": "StrongPass123" }
