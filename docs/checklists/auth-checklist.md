# Чек-лист: Функциональное тестирование API авторизации

**Модуль:** Авторизация
**Инструмент:** Postman 12.7.0
**Окружение:** Local
**OS:** Windows 10 Pro

---

## POST /register

| # | Проверка | Результат |
|---|----------|-----------|
| 1 | Регистрация с валидными данными — 201 Created | PASS |
| 2 | Тело ответа содержит id пользователя | FAIL |
| 3 | Пустой body {} — 400 Bad Request | PASS |
| 4 | Нет поля name — 400 + описание ошибки | PASS |
| 5 | Нет поля password — 400 + описание ошибки | PASS |
| 6 | Пароль короче минимальной длины — 400 | PASS |
| 7 | Пароль = null — 400 | PASS |
| 8 | name длиннее 20 символов — 400 | FAIL |
| 9 | Очень длинный пароль (1000+ символов) — не 500 | PASS |
| 10 | Повторная регистрация с тем же name — 400 | PASS |

## POST /login

| # | Проверка | Результат |
|---|----------|-----------|
| 1 | Логин с корректными данными — 200 OK | PASS |
| 2 | Ответ содержит access_token и refresh_token в куках | PASS |
| 3 | Токены имеют правильную JWT структуру (проверено на jwt.io) | PASS |
| 4 | Пустой body {} — 401 | PASS |
| 5 | Нет поля name — 401 | PASS |
| 6 | Нет поля password — 401 | PASS |
| 7 | Пустая строка name — 401 | PASS |
| 8 | password = null — 401 | PASS |
| 9 | Неверный пароль — 401 (не 403, не 500) | PASS |
| 10 | Несуществующий name — 401 (идентично неверному паролю) | PASS |
| 11 | Payload JWT содержит user_id, exp, iat | PASS |
| 12 | Payload JWT не содержит пароль и чувствительные данные | PASS |

## Найденные дефекты

| ID | Описание | Severity |
|----|----------|----------|
| BUG-001 | Отсутствует валидация максимальной длины поля name | Major |
| BUG-002 | Пароль не хешируется в БД при создании через админку | Critical |
| BUG-003 | Неверный body ответа при успешной регистрации | Minor |
| BUG-004 | Поле timestamp сохраняется без временной зоны | Major |
