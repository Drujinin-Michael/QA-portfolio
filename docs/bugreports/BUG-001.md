# BUG-001 — Отсутствует валидация максимальной длины поля name

**ID:** DEV-10
**Статус:** К выполнению
**Severity:** Major
**Priority:** High

## Окружение

| Параметр | Значение |
|----------|----------|
| Окружение | Local |
| Метод | POST |
| URL | http://127.0.0.1:8000/api/auth/register |
| Инструмент | Postman 12.7.0 |
| OS | Windows 10 Pro |

## Описание

Согласно документации поле name должно принимать максимум 20 символов.
При передаче строки длиннее 20 символов сервер не возвращает ошибку валидации,
а создаёт пользователя успешно.

## Шаги воспроизведения

1. Открыть Postman
2. Создать POST-запрос на http://127.0.0.1:8000/api/auth/register
3. Заголовок: Content-Type: application/json
5. Нажать Send

## Ожидаемый результат

- Статус-код: 400 Bad Request или 422 Unprocessable Entity
- Body содержит сообщение об ошибке валидации поля name

## Фактический результат

- Статус-код: 201 Created
- Пользователь создаётся с именем длиннее 20 символов
- Валидационная ошибка не возвращается

## Граничные значения

| Длина name | Результат | Ожидается |
|------------|-----------|-----------|
| 19 символов | 201 Created | 201 Created |
| 20 символов | 201 Created | 201 Created |
| 24 символа | 201 Created | 400/422 |
| 50 символов | 201 Created | 400/422 |
