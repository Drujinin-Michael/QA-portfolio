# BUG-004 — Поле timestamp сохраняется без временной зоны

**ID:** DEV-17
**Статус:** К выполнению
**Severity:** Major
**Priority:** Medium

## Окружение

| Параметр | Значение |
|----------|----------|
| Окружение | Local |
| Метод | POST |
| URL | http://127.0.0.1:8000/api/auth/login |
| Инструмент | Postman 12.7.0 + PgAdmin |
| OS | Windows 10 Pro |

## Описание

После входа пользователя поле last_login сохраняется в БД без информации о временной зоне. Ожидается формат TIMESTAMP WITH TIME ZONE.

## Шаги воспроизведения

1. Открыть Postman
2. POST на http://127.0.0.1:8000/api/auth/login
3. Body: { "name": "testuser", "password": "StrongPass123" }
4. Нажать Send, получить 200 OK
5. Открыть PgAdmin и найти запись пользователя
6. Проверить поле last_login

## Ожидаемый результат

- Поле last_login имеет тип TIMESTAMP WITH TIME ZONE
- Значение содержит timezone-offset, например: 2026-04-22T14:30:00+05:00

## Фактический результат

- Поле last_login сохраняется как NULL
- Timezone-offset отсутствует

## Потенциальные последствия

| Сценарий | Последствие |
|----------|-------------|
| Пользователи из разных часовых поясов | Некорректное отображение времени входа |
| Сортировка по дате входа | Ошибки при сравнении timestamp-значений |
| Логирование и аудит | Невозможно точно определить реальное время события |
