# BUG-002 — Пароль не хешируется в БД при создании через админку

**ID:** DEV-15
**Статус:** К выполнению
**Severity:** Critical
**Priority:** High

## Окружение

| Параметр | Значение |
|----------|----------|
| Окружение | Local |
| URL | http://127.0.0.1:8000/admin/ |
| Инструмент | Django Admin + PgAdmin |
| OS | Windows 10 Pro |

## Описание

При создании пользователя через Django Admin пароль сохраняется в базе данных в открытом виде. Через API регистрации пароль хешируется корректно.

## Шаги воспроизведения

1. Открыть http://127.0.0.1:8000/admin/
2. Перейти в Users и нажать Add
3. Заполнить поля name и password, нажать Save
4. Открыть PgAdmin и найти запись пользователя
5. Проверить колонку password

## Ожидаемый результат

- В колонке password хранится хешированное значение (например pbkdf2_sha256...)
- Пароль в открытом виде не отображается

## Фактический результат

- В колонке password виден пароль в открытом виде
- Хеширование не применяется
